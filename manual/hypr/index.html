Hyprland

Hypr Config Reference

Leenium uses a layered Hyprland setup. This page explains which files are project defaults, which files are yours to edit, and which helper commands can overwrite them.

Load Order

Your main ~/.config/hypr/hyprland.conf first loads Leenium defaults from ~/.local/share/leenium/default/hypr, then loads user overrides from ~/.config/hypr.

Defaults: autostart, media / clipboard / tiling / utilities bindings,
envs, looknfeel, input, windows, current theme.

Overrides: monitors.conf, input.conf, bindings.conf,
looknfeel.conf, autostart.conf.

Practical rule: edit ~/.config/hypr. Treat ~/.local/share/leenium/default/hypr as project-owned baseline config.

User Override Tree

Main overrides

  • monitors.conf
  • input.conf
  • bindings.conf
  • looknfeel.conf
  • autostart.conf

Related runtime files

  • hypridle.conf
  • hyprlock.conf
  • hyprsunset.conf
  • xdph.conf

Leenium Default Tree

  • looknfeel.conf, input.conf, windows.conf, envs.conf define baseline behavior
  • autostart.conf starts base services and first-run entrypoints
  • bindings/ holds media, clipboard, tiling, and utility bindings
  • apps.conf and apps/*.conf hold app-specific window rules
  • plain-bindings.conf exists for simpler direct binding overrides

Useful Commands While Tweaking

  • hyprctl reload
  • leenium-menu-keybindings --print
  • leenium-reinstall-configs

leenium-reinstall-configs is not a light refresh. It is a reset-style tool and can overwrite local customizations.
