Leenium Troubleshooting

When something breaks, start with the shortest signal path: logs first, smallest safe recovery command second, broad resets last.

Install Failures

Always start here for install issues:

/var/log/archinstall/install.log
/var/log/leenium-install.log

If you need to share the failure:

leenium-upload-log install

First-Run Problems

If the system boots but first-login automation did not complete, try:

leenium-cmd-first-run

Then verify the first-run marker state:

~/.local/state/leenium/first-run.mode

UI Component Recovery

Restart services

  • leenium-restart-waybar
  • leenium-restart-walker
  • leenium-restart-mako

Reapply config

  • leenium-reinstall-configs

leenium-reinstall-configs is a stronger reset tool than the service restarts above. Use it when config drift is the likely cause, not as the first response to every UI problem.

Update Recovery

  • leenium-update for the normal update path
  • leenium-update-analyze-logs when update logs mention initramfs or related warnings
  • leenium-reinstall when the baseline package state needs to be forced back into line

Collect Debug Information

  • leenium-debug
  • leenium-debug --print
  • leenium-upload-log this-boot
  • leenium-upload-log last-boot

Recovery Order That Usually Works

  1. read the relevant logs
  2. run the smallest targeted recovery command
  3. collect debug output before broader resets
  4. only then use reinstall-style commands
