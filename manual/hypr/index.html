Load Order (Important)

Your main ~/.config/hypr/hyprland.conf first loads defaults from ~/.local/share/leenium/default/hypr, then loads user overrides from ~/.config/hypr.

Defaults: autostart, media/clipboard/tiling/utilities bindings,
envs, looknfeel, input, windows, current theme.

Overrides: monitors.conf, input.conf, bindings.conf,
looknfeel.conf, autostart.conf.

Practical rule: edit files in ~/.config/hypr; keep default/hypr as baseline.

User Config Tree (`leenium/config/hypr`)

  • hyprland.conf: source order and entry point.
  • monitors.conf: display layout and scaling.
  • bindings.conf: personal keybind overrides.
  • input.conf: keyboard, touchpad, pointer settings.
  • looknfeel.conf: local aesthetic overrides.
  • autostart.conf: startup programs and services.
  • hypridle.conf, hyprlock.conf, hyprsunset.conf, xdph.conf: lock/idle/nightlight/portal behavior.

Default Config Tree (`leenium/default/hypr`)

  • looknfeel.conf, input.conf, windows.conf, envs.conf: baseline session behavior.
  • autostart.conf: base daemons + leenium-cmd-first-run.
  • bindings/:
  • media.conf: XF86 volume/brightness/media keys.
  • clipboard.conf: universal copy/paste/cut + clipboard manager.
  • tiling-v2.conf: modern window/workspace/group controls.
  • utilities.conf: menus, notifications, toggles, capture, sharing.
  • apps.conf + apps/*.conf: app-specific window rules (browser, terminals, telegram, steam, qemu, etc.).
  • plain-bindings.conf and root bindings.conf: compatibility/deprecated binding layers.

Useful Commands While Tweaking

  • hyprctl reload or leenium-restart-hyprctl
  • leenium-menu-keybindings --print to inspect live binds
  • leenium-reinstall-configs to restore defaults (overwrites local config)
