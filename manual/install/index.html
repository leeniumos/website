Scope

Leenium is the post-archinstall system layer. The main entrypoint is install.sh, designed to run from ~/.local/share/leenium in the ISO/install workflow.

Requirements Enforced in Preflight

  • Vanilla Arch Linux (not CachyOS, EndeavourOS, Garuda, Manjaro).
  • Run as normal user, not root.
  • x86_64 architecture.
  • Secure Boot disabled.
  • No preinstalled GNOME/KDE desktop stack.
  • limine bootloader is present.
  • Root filesystem is btrfs.

Install Stages

  1. preflight: guard checks, log setup, migration run, first-run mode marker, mkinitcpio hook deferral.
  2. packaging: base, fonts, neovim, icons, webapps, TUIs, and hardware package sets.
  3. config: copy defaults, theme setup, shell and app config, hardware fixes.
  4. login: Plymouth, SDDM, Limine/Snapper/mkinitcpio integration.
  5. post-install: hibernation setup, pacman cleanup, reboot permission, final screen.

Logs and Success Markers

  • Installer log: /var/log/leenium-install.log
  • Base install log: /var/log/archinstall/install.log
  • Completion marker in chroot mode: /var/tmp/leenium-install-completed

Every stage script is executed through logged wrappers, so failed scripts are timestamped in the install log.
