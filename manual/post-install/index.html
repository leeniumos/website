Leenium Post-Install Guide

This chapter covers what Leenium does automatically on first login, what you should do next, and which optional setup commands matter after the system is already usable.

What Runs Automatically

On login, Hyprland autostart launches leenium-cmd-first-run once when this marker exists:

~/.local/state/leenium/first-run.mode

System setup

  • power profile selection
  • battery monitor timer setup
  • UFW defaults and Docker integration
  • /etc/resolv.conf resolver setup

Desktop cleanup

  • GNOME interface defaults
  • icon cache refresh
  • Elephant user service activation
  • installer sudoers cleanup and critical notifications

Best Immediate Actions

  1. Connect networking if needed with leenium-launch-wifi
  2. Run a full update with leenium-update
  3. Open the keybindings cheatsheet with Super + K
  4. Check theme and font state with leenium-theme-list and leenium-font-list

If the machine feels “mostly installed but slightly off,” run the update before changing anything else. That removes a lot of false alarms from stale package state.

If First Boot Did Not Finish

The fastest manual retry path is:

leenium-cmd-first-run

Then continue with the Troubleshooting chapter if the result still looks wrong.
