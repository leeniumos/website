Scope

Leenium is the post-archinstall system layer. The main entrypoint is install.sh, designed to run from ~/.local/share/leenium in the ISO install flow.

This page describes the expected install behavior. If the code and this page ever disagree, the docs should be treated as stale and updated immediately.

Preflight Requirements

The installer assumes all of the following before continuing:

System assumptions

  • Vanilla Arch Linux
  • x86_64 architecture
  • btrfs root filesystem
  • limine bootloader present

Safety assumptions

  • Run as a normal user, not root
  • Secure Boot disabled
  • No preinstalled GNOME or KDE stack
  • No derivative distro state like Manjaro or Garuda

Install Stages

  1. preflight: guard checks, logging, migrations, first-run marker prep, and mkinitcpio hook deferral
  2. packaging: base packages, fonts, Neovim, icons, webapps, TUIs, and hardware-specific package sets
  3. config: copy defaults, apply theme setup, deploy app and shell config, and run hardware fixes
  4. login: Plymouth, SDDM, Limine, Snapper, and mkinitcpio integration
  5. post-install: hibernation setup, pacman cleanup, reboot permissions, and final completion state

Logs and Completion Markers

The first response to an install issue should be log review, not trial-and-error changes.

/var/log/leenium-install.log
/var/log/archinstall/install.log
/var/tmp/leenium-install-completed

Every stage script is run through logging wrappers. If a stage fails, the install log should show the failed script with useful timestamps. Start there.

Fast Triage

  • If install never starts, check preflight assumptions first.
  • If packages fail, inspect /var/log/archinstall/install.log.
  • If Leenium-specific config fails, inspect /var/log/leenium-install.log.
  • If the install finished but first login feels wrong, continue with the Post-Install Guide.
