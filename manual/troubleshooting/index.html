Troubleshooting

Common failures and direct recovery paths from Leenium tooling.

Manual Home Website Home

Install Failures

Check logs first:

/var/log/archinstall/install.log
/var/log/leenium-install.log

If needed, share logs with:

leenium-upload-log install

First-Run Did Not Complete

If first boot tasks did not run, manually invoke:

leenium-cmd-first-run

Verify marker file state:

~/.local/state/leenium/first-run.mode

UI Component Issues

  • Reload Hyprland config: leenium-restart-hyprctl
  • Restart Waybar: leenium-restart-waybar
  • Restart Walker: leenium-restart-walker
  • Restart notifications: leenium-restart-mako
  • Re-copy defaults: leenium-reinstall-configs

Update Problems

  • Standard update: leenium-update
  • Analyze update log for initramfs warning: leenium-update-analyze-logs
  • Force reinstall baseline: leenium-reinstall

Collect Debug Information

  • Interactive debug report: leenium-debug
  • Print-only debug report: leenium-debug --print
  • Boot log upload: leenium-upload-log this-boot or leenium-upload-log last-boot
